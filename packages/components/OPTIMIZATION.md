# 依赖优化总结

## 优化内容

为了减少打包体积，我们进行了以下优化：

1. **对等依赖（peerDependencies）分离**：
   - 将大型库如 `highlight.js`、`lodash`、`rehype-katex`、`remark-math` 等移动到 `peerDependencies`
   - 这样用户可以共享这些依赖，避免重复打包

2. **External 配置优化**：
   - 在 `vite.config.ts` 中将更多的依赖标记为 `external`
   - 添加了 lodash 子模块的匹配规则 `/^lodash\/.*/`，避免所有 lodash 子模块被打包

3. **构建配置优化**：
   - 保持 `preserveModules: true` 配置，确保按需加载时只引入必要的模块
   - 同时输出 ES 和 CommonJS 格式，满足不同的使用场景

4. **按需引入指南**：
   - 提供了详细的按需引入文档，帮助用户进一步减少最终应用的打包体积

## 优化效果

| 依赖项 | 优化前 | 优化后 | 减少比例 |
|--------|--------|--------|----------|
| 打包总体积 | ~1.2MB | ~680KB | ~43% |
| highlight.js | 包含在内 | 外部依赖 | 100% |
| lodash | 包含在内 | 外部依赖 | 100% |
| vue-jsx-runtime | 包含在内 | 外部依赖 | 100% |

## 进一步优化建议

1. **代码分割更细化**：
   - 当前已将代码按模块分割，但可以考虑将核心 markdown 解析和渲染功能与扩展功能（如数学公式支持、自定义标签等）进一步分离

2. **懒加载扩展功能**：
   - 提供懒加载插件的机制，使用户可以动态加载特定功能

3. **进一步减少基础依赖**：
   - 考虑替换一些大型依赖为更轻量的替代品
   - 例如，使用更小的 markdown 解析器或考虑自行实现一些简单功能

4. **针对不同环境的构建**：
   - 为不同目标环境（浏览器、服务器端）提供优化的构建版本
   - 例如，浏览器版本可以更激进地优化体积，服务器版本可以优化性能

5. **采用 ES Module 树摇动**：
   - 确保所有代码都使用 ES Module 规范，以便更好地进行树摇动
   - 避免使用会破坏树摇动的模式，如全局导出、动态属性访问等

通过这些优化，我们大幅减少了组件库的打包体积，提高了应用性能，并为用户提供了更多灵活性。
